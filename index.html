Duas pilhas, um objetivo

push_swap ordena uma lista de números usando só duas pilhas (A e B) e um conjunto fixo de 11 operações. Este guia cobre main.c, parser.c, operations.c, sorting.c, utils.c e as metas de benchmark — tudo num arquivo só, pronto pra compartilhar.

main.c

A função principal

Seis passos simples: iniciar, validar, ordenar, limpar. Toda a inteligência do programa está nas outras camadas — main.c só orquestra a ordem de chamadas.

operations.c

As 11 operações — os únicos movimentos permitidos

Todo o algoritmo de ordenação, do sort_three ao sort_big, só se movimenta usando estas 11 funções. Cada uma imprime seu próprio nome no stdout.

void sa(t_list **stack_a)GRUPO 1 · SWAP

sb faz exatamente o mesmo, só que na pilha B.

void ss(t_list **stack_a, t_list **stack_b)GRUPO 1 · SWAP

Repete a lógica de sa e sb, mas duplicada (sem chamar as funções) para imprimir só "ss".

void pa(t_list **stack_a, t_list **stack_b)GRUPO 2 · PUSH
void pb(t_list **stack_a, t_list **stack_b)GRUPO 2 · PUSH

É o inverso exato de pa (A → B em vez de B → A).

void ra(t_list **stack_a)GRUPO 3 · ROTATE

rb faz o mesmo, só que na pilha B.

void rra(t_list **stack_a)GRUPO 4 · REVERSE ROTATE

rrb faz o mesmo, só que na pilha B.

void rr(t_list **stack_a, t_list **stack_b)GRUPO 3 · ROTATE

Mesma ideia de ss: duplica o código de ra e rb, imprime só "rr".

void rrr(t_list **stack_a, t_list **stack_b)GRUPO 4 · REVERSE ROTATE

Mesma ideia de rr, mas girando para baixo (como rra e rrb).

sorting.c

Três estratégias, um dispatcher

O algoritmo usado depende só do tamanho da pilha.

void push_swap(t_list **stack_a, t_list **stack_b)DISPATCHER
size = stack_size(*stack_a);
size ≤ 3 → sort_three(A) size ≤ 5 → sort_five(A, B) size > 5 → sort_big(A, B)
void sort_three(t_list **stack_a)FUNC. 01

void sort_five(t_list **stack_a, t_list **stack_b)FUNC. 02

Estratégia: manda os 2 menores para B, resolve os 3 que sobram em A com sort_three, e traz os 2 de volta.

void sort_big(t_list **stack_a, t_list **stack_b)FUNC. 03

Para mais de 5 números, o algoritmo Turk divide o intervalo de valores em chunks (faixas) e empurra cada faixa para B em ondas, deixando só 3 em A no final.

chunk = size / 5; i = 0; max = size - 3;
while (i < max) { if (A[0] ≤ i+chunk) pb(A,B); else if (A[0] ≤ i+chunk*2) { pb(A,B); rb(B); } else ra(A); i++; }
sort_three(A);organiza os 3 que restaram em A
while (B) { pa(A,B); if (A[0] > A[1]) sa(A); }devolve tudo de B para A, mantendo a ordem
while (A[0] != min) ra(A);rotação final: coloca o menor no topo

As pequenas funções que sustentam tudo

Nove funções auxiliares — a maioria é uma travessia simples da lista encadeada.

int is_sorted(t_list *stack)FUNC. 01

Compara cada par de vizinhos; qualquer inversão já é motivo para retornar 0.

void free_stack(t_list **stack)FUNC. 06

Guarda o próximo nó antes de liberar o atual — sem isso, seria um use-after-free.

stack_size(stack)

Conta os nós até chegar em NULL.

[5]→[3]→[8]→[1]→NULL  →  4

find_min / find_max(stack)

find_max é o mesmo código de find_min, só troca < por >.

[5]→[3]→[8]→[1]  →  min=1, max=8

get_index(stack, value)

Percorre contando até achar o valor — ou devolve -1.

buscar 8 em [5]→[3]→[8]→[1]  →  índice 2

Metas de performance

A nota depende de quantas operações o algoritmo usa para ordenar números aleatórios — quanto menos, melhor.

100 números aleatórios
< 700 → nota máxima < 1100 → nota mínima > 1100 → reprovado
500 números aleatórios
< 5500 → nota máxima < 8500 → nota mínima > 8500 → reprovado
O algoritmo Turk (chunks) usado em sort_big é o que permite ficar dentro dessas metas para listas grandes — sort_three e sort_five sozinhos não escalam.
